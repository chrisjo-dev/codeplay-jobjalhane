# 기획
**"데이터 기반의 증거(적성검사)를 활용하여, 커리어 탐색의 불확실성을 겪는 사회초년생에게 가장 신뢰할 수 있는 맞춤형 직업과 로드맵을 제시하는 세계 최고 수준의 AI 솔루션"**을 구축하는 것입니다.

이 기획의 핵심 맥락을 세 부분으로 나누어 설명할 수 있습니다.

1. 타겟과 문제 정의 (The 'Why')
타겟: 사회초년생 (어떤 직업을 가져야 할지 확신이 없는 사람들)
문제: 기존의 직업 추천은 너무 일반적이거나, 트렌드에 편중되거나, 개인의 특성을 깊이 있게 반영하지 못합니다. 특히, 적성검사를 받아도 그 결과를 실제 직업과 연결 짓기 어렵다는 '단절(gap)'이 존재합니다.

2. 핵심 솔루션 (The 'What')
'근거 기반' 분석: 사용자가 직접 신뢰할 수 있는 외부 적성검사 결과지(PDF)를 '증거'로 제출하게 합니다.

AI의 역할:
분석: PDF의 복잡한 텍스트와 데이터를 AI가 정밀하게 분석하여 사용자의 핵심 적성, 흥미, 강점을 추출합니다.
매칭: 단순 키워드 매칭이 아닌, 추출된 프로필과 '정확하게 부합하는' 최적의 직업군을 추천합니다.
제시: 해당 직업을 갖기 위한 구체적이고 실행 가능한 '커리어 로드맵'을 생성하여 제공합니다.

3. 차별점이자 목표 (The 'How' - World-Class)
'세계 최고 수준'의 의미:
정확성: AI가 PDF를 분석하고 직업을 추천하는 과정이 감(感)이 아닌 데이터에 기반하여, 그 '근거'를 명확히 제시해야 합니다.
신뢰성: "왜 이 직업이 당신에게 맞는지"에 대한 설득력이 기존 솔루션과 비교할 수 없을 정도로 높아야 합니다.
실행 가능성: 로드맵이 누구나 아는 뻔한 내용이 아니라, 개인의 현재 상태(적성검사 기반)에서 시작하는 '초개인화'된 경로여야 합니다.


# MVP 핵심 기능 4가지
1. (제한된) PDF 업로드 및 텍스트 추출
'세계 최고' 버전은 어떤 PDF든 분석하지만, MVP는 범위를 극단적으로 좁혀야 합니다.

구현:
[Must] 사용자가 PDF 파일을 업로드하는 기본 인터페이스.
[MVP] 모든 PDF가 아닌, 특정 1~2개의 유명 적성검사 결과지 형식만 지원한다고 명시합니다. (e.g., "현재 OOO 검사, XXX 검사 결과지만 지원합니다.")
[기술] 복잡한 멀티모달 AI 대신, 텍스트가 복사되는(text-selectable) PDF를 가정하고 PyPDF2나 pdfplumber 같은 간단한 Python 라이브러리로 텍스트를 먼저 추출합니다.

2. LLM 기반 핵심 적성 프로파일링
여러 에이전트(분석관, 해석가) 대신, 강력한 LLM에게 '프롬프트 엔지니어링'으로 역할을 한 번에 맡깁니다.

구현:
[Must] 1번에서 추출한 PDF 텍스트를 LLM(e.g., GPT-4o, Claude 3) API로 전송합니다.
[MVP] 매우 구체적인 **'단일 프롬프트'**를 설계합니다.
"너는 20년 차 커리어 카운슬러야. 다음은 OOO 적성검사 결과지 텍스트야. [텍스트 삽입]. 여기서 사용자의 1) 핵심 강점 3가지, 2) 흥미 분야 2가지, 3) 보완이 필요한 약점 1가지를 요약해서 JSON 형식으로 추출해 줘."
[결과] LLM이 사용자의 '정형화된 프로필' (JSON)을 생성합니다.

3. 'Mini-DB' 기반 직업 추천 (가장 중요)
실시간 '커리어 KG'와 GNN은 MVP 단계에서 불가능합니다. 직접 만든 소규모 고품질 직업 DB가 핵심입니다.

구현:

[Must] 수동으로 100~200개 정도의 핵심 직업 데이터베이스('Mini-DB')를 구축합니다. (e.g., SQL DB 또는 간단한 JSON 파일)
[DB 구조] { "job_name": "프론트엔드 개발자", "description": "...", "required_aptitudes": ["논리력", "창의성", "꼼꼼함"], "related_interests": ["IT", "디자인"], ... }
[MVP] 2번에서 생성된 사용자의 '프로필 JSON'과 'Mini-DB'의 'required_aptitudes' 및 'related_interests'를 **벡터 임베딩(Vector Embedding)**하여 유사도를 검색합니다.
[기술] OpenAI Embeddings API로 프로필과 직업 설명을 벡터화하고, Pinecone 또는 FAISS (로컬) 같은 벡터 DB에서 가장 유사한 Top 3 직업을 찾습니다.

4. LLM 기반 '범용' 로드맵 생성
RAG(검색 증강 생성)나 동적 생성이 아닌, LLM의 **'일반 지식'**을 활용해 로드맵을 즉시 생성합니다.

구현:

[Must] 사용자가 3번에서 추천된 직업(e.g., '프론트엔드 개발자')을 선택합니다.
[MVP] 해당 직업명을 LLM에 다시 프롬프트로 전달합니다.
"사회 초년생이 '프론트엔드 개발자'가 되기 위한 3단계 핵심 로드맵을 생성해 줘. 1단계(기초), 2단계(심화/기술), 3단계(포트폴리오)로 나누고, 각 단계별로 배워야 할 핵심 키워드 2~3개만 알려줘."
[결과] LLM이 생성한 '일반적이지만 그럴듯한' 로드맵을 사용자에게 바로 보여줍니다.


기술 스택 
Frontend	React	빠르고 반응성 높은 UI/UX
Backend	Python (FastAPI)	AI/ML 연동 용이성 및 고성능
PDF 처리	Google Document AI, LayoutLMv3	레이아웃/표/그래프를 포함한 PDF 분석
LLM/Generative	OpenAI API (GPT-4o), Anthropic (Claude 3)	에이전트 시스템, RAG, 로드맵 생성
지식 그래프	Neo4j 또는 Amazon Neptune	'커리어 지식 그래프' 구축 및 쿼리
GNN	PyTorch Geometric (PyG)	그래프 기반의 복합적 관계 추론
벡터 DB	Pinecone, Weaviate, Milvus	초고속 직업-적성 유사도 검색
인프라/배포	Docker, Kubernetes (EKS/GKE)	마이크로서비스 아키텍처, 자동 확장성
CI/CD	GitHub Actions, Jenkins	신속하고 안정적인 빌드 및 배포
